# Portfolio Framework

## 这套框架解决什么问题

`portfolio/` 这套框架不是给“单只股票择时”用的，而是给“组合轮动 + 收盘后出计划 + 次日执行”这种场景用的。

核心目标是：

- 每天收盘后，先在一个股票池里做筛选和打分
- 决定下一交易日应该买什么、卖什么
- 支持输入当前持仓和可用现金
- 输出组合回测结果和下一交易日委托建议

当前第一版只实现了一个股票池：

- `hs300`：沪深300

当前第一版只实现了一个组合策略：

- `hs300_rotation`：沪深300日频趋势轮动

## 框架结构

### 1. `service.py`

负责组合回测的“调度层”，主要做这些事情：

- 接收回测请求和明日计划请求
- 读取沪深300成分股
- 读取这些股票的日线行情
- 把数据整理成策略方便消费的结构
- 调用对应策略的 `runner` 或 `planner`
- 统一拼装返回结果

可以理解成：它不决定买什么卖什么，它只负责把数据喂给策略，然后把结果返回给前端。

### 2. `registry.py`

负责组合策略注册。

里面定义了：

- 默认策略 ID
- 默认股票池 ID
- 支持的回测周期
- 支持的股票池
- 所有组合策略的注册表

以后如果新增新的组合策略，只要在这里注册，就能被前端自动发现。

### 3. `strategies/hs300_rotation.py`

这是当前真正的“选股 + 调仓逻辑”。

它包含两部分：

- `run_backtest()`：历史回测
- `build_plan()`：生成下一交易日计划

### 4. `utils.py`

放公共函数，比如：

- 年化收益计算
- 资金/价格安全转换
- 整手股数处理
- 下一工作日推算

## 轮动策略的人话解释

这套策略可以理解成：

> 每天收盘后，在沪深300里找“当前趋势最好、动量最好、形态最顺”的几只股票，优先持有它们；  
> 如果已有持仓还够强，就尽量继续拿着；  
> 如果明显走弱或者掉出目标组合，就准备在下一交易日卖出。

它不是抄底策略，而是“趋势轮动策略”。

也就是说，它不追求买在最低点，而是希望：

- 只做已经走强的股票
- 尽量让资金一直待在相对强势的股票里

## 每天收盘后，系统具体做什么

### 第一步：确定股票池

当前股票池固定为沪深300。

也就是说，不会在全市场乱选，只会在沪深300成分股里做轮动。

这样做有几个好处：

- 成分股整体流动性更好
- 基本面质量通常更稳定
- 更适合“收盘后计划、次日挂单”的执行方式

### 第二步：做基础过滤

系统不会给所有股票打分，而是先做一轮硬过滤。

一只股票必须同时满足这些条件，才有资格进入候选名单：

1. 不是 ST，且没有退市风险
2. 上市时间足够长
3. 收盘价高于 `MA20`
4. `MA20` 高于 `MA60`
5. `20日涨幅 > 0`
6. `MACD 柱线 > 0`
7. `RSI` 落在设定区间内

翻成人话就是：

- 不做明显有问题的股票
- 不做刚上市没多久的不稳定股票
- 不做趋势向下的股票
- 只做已经处于中短期上升趋势中的股票

## 候选股票怎么打分

通过基础过滤之后，系统会给每只股票打一个综合分数。

当前分数由 5 部分组成：

### 1. 动量分

核心看：

- `20日涨幅`

涨得越强，动量分越高。

但不是无限加分，会做截断，避免极端股票分数过度失真。

### 2. 趋势分

核心看：

- `收盘价 / MA20`

如果收盘价明显站在 `MA20` 上方，说明短期趋势更强。

### 3. 结构分

核心看：

- `MA20 / MA60`

如果 `MA20` 比 `MA60` 高得更明显，说明中期趋势结构更漂亮。

### 4. RSI 分

不是 RSI 越高越好。

当前逻辑更偏向：

- RSI 靠近 `60` 左右最舒服

因为：

- 太低说明还弱
- 太高说明可能过热

所以它更像在找“强，但还没极端拥挤”的股票。

### 5. MACD + 流动性分

核心看：

- MACD 是否继续走强
- 换手率是否处在比较正常的区间

策略不希望买到：

- 动能已经衰减的票
- 或者流动性太差、不适合提前挂单的票

## 怎么决定最终买哪几只

股票打分后，会按分数从高到低排序。

如果你设置：

- 最大持仓数 = 3

那系统最终目标组合一般就是：

- 得分最高的前 3 只

但这里有一个“持仓缓冲名次”参数：

- `keep_buffer`

它的作用是：

- 如果你现在已经持有某只股票
- 它虽然不是前 3 名了
- 但仍然排在前 `3 + keep_buffer` 名以内

那系统会倾向于继续保留它，而不是立刻换掉。

这能减少不必要的频繁换手。

## 为什么不是每天都换仓

因为这套策略还加了两个约束：

### 1. 最短持有天数

- `min_holding_days`

意思是：

- 一只股票买进去之后
- 即使它掉出目标组合
- 也不会立刻卖
- 只有持有时间超过最短持有天数后，才允许因为“排名掉队”被换掉

这样做是为了减少“今天买、明天卖”的噪音交易。

### 2. 硬退出条件

即使没到最短持有天数，只要出现明显走弱，也可以直接退出。

当前硬退出条件包括：

1. 收盘价明显跌破 `MA20`
2. `MA20 <= MA60`
3. `MACD 柱线 <= 0`
4. 跌破成本止损线

这代表：

- 对“轻微排名波动”会宽容一点
- 但对“趋势真的坏了”不会死扛

## 回测里怎么执行买卖

### 信号时点

信号在 `T 日收盘后` 产生。

也就是说，系统用的是：

- 当天收盘后的指标和排名

### 执行时点

交易在 `T+1 日开盘` 执行。

这很重要，因为这更贴近你的真实使用方式：

- 晚上看结果
- 第二天开盘前挂单
- 盘中不再盯盘

### 买入逻辑

如果有空仓位，就从候选排名高到低补齐。

每只股票大致按这个思路分配资金：

- 先计算组合总资产
- 预留一部分现金
- 剩余资金平均分给持仓数

例如：

- 最大持仓 3 只
- 现金保留比例 10%

那么大概会把 90% 的资金均分给 3 只股票。

### 卖出逻辑

如果某只股票不该继续持有了，就在下一交易日开盘卖掉。

回测中使用的是：

- 下一交易日开盘价

## 明日计划是怎么生成的

`build_plan()` 的逻辑和回测的信号逻辑是一致的。

区别是：

- 回测面对的是“历史某一天的持仓”
- 明日计划面对的是“你现在实际输入的持仓和现金”

你可以输入：

- 当前现金
- 当前持仓股票
- 持仓股数
- 持仓成本

系统会输出三类信息：

### 1. 目标组合

下一交易日理论上最应该持有哪几只股票。

### 2. 保留持仓

哪些股票虽然你已经持有，但它们依然在目标组合内，所以建议继续持有。

### 3. 下一交易日动作

给出建议委托清单：

- 卖出哪些股票
- 买入哪些股票
- 每只建议多少股
- 建议参考价格
- 每笔动作的原因

## 参考价格怎么来的

当前第一版不是直接预测“明天开盘价”，而是用今天收盘价做一个保守偏移。

### 卖出参考价

- `收盘价 * (1 - sell_limit_buffer)`

也就是：

- 比今天收盘价略低一点

这样更容易成交。

### 买入参考价

- `收盘价 * (1 + buy_limit_buffer)`

也就是：

- 比今天收盘价略高一点

这样是为了提高买入成交概率。

它本质上是一种“委托价建议”，不是对真实开盘价的预测。

## 你在页面里看到的结果怎么理解

### 1. 收益指标

- `总收益率`：整个回测区间赚了多少
- `年化收益`：把这段收益按全年复利折算
- `最大回撤`：历史最大浮亏幅度
- `胜率`：卖出交易里有多少笔是赚钱的

### 2. 最新候选排名

表示：

- 如果回测结束那天再做一次选股
- 当前最强的股票是谁
- 为什么入选

### 3. 调仓节点

表示：

- 哪一天收盘后发出信号
- 下一交易日执行了哪些买卖

### 4. 交易明细

表示：

- 每一笔买卖的具体执行情况
- 包括数量、价格、金额、原因、收益

### 5. 明日计划

表示：

- 如果今天就是最新交易日
- 明天应该怎么处理你的仓位

## 关键参数怎么理解

### `score_threshold`

最低得分门槛。

越高：

- 候选股票越少
- 质量要求越高
- 但可能错过一些中等强度机会

### `min_rsi` / `max_rsi`

限制 RSI 区间。

作用是：

- 不买太弱的
- 也尽量不追太热的

### `min_listed_days`

限制最短上市时间。

作用是：

- 避免新股和次新股带来的不稳定性

### `keep_buffer`

持仓缓冲名次。

越大：

- 越不容易因为短期排名变化被换出
- 换手率通常会下降

### `min_holding_days`

最短持有天数。

越大：

- 越像“波段持有”
- 不容易频繁换仓

### `stop_loss_pct`

成本止损比例。

作用是：

- 即使一只股票曾经很强
- 只要买入后跌得太多，也会触发退出

### `buy_limit_buffer` / `sell_limit_buffer`

用于生成次日挂单的参考价格。

它们只影响“计划价格”，不改变核心选股逻辑。

## 这套策略最适合什么人

比较适合：

- 没时间盯盘
- 只能在收盘后看结果
- 想在次日开盘前一次性挂单
- 更偏向做强势趋势轮动的人

## 当前版本的已知局限

当前是第一版，重点是把流程跑通。

所以还存在一些现实层面的简化：

1. 沪深300成分股目前使用的是“当前成分”，不是历史动态成分
2. 还没有计入手续费、印花税、滑点
3. 下一交易日目前按工作日推算，不是严格交易日历
4. 明日计划中的委托价格是“参考价”，不是真实开盘价预测

## 后续如果你要继续优化

优先建议做这几件事：

1. 接入真实交易日历
2. 加入手续费和滑点
3. 支持历史成分股变化
4. 增加行业约束和单行业集中度控制
5. 增加“黑名单”或“持仓白名单”

## 一句话总结

这套策略不是在猜最低点，而是在做一件更适合非盯盘用户的事情：

> 每天收盘后，从沪深300里选出当前趋势最好的少数股票，  
> 尽量把仓位留在强势股里，  
> 弱了就换，坏了就卖，  
> 并把这些动作提前转成第二天可以直接委托的计划。
